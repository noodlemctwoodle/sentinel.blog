## Home Assistant Configuration

First of all, you will need to create a Long-Lived Access Token in Home Assistant, to do this click on your Username in the lower-left corner of your browser and then scroll to the bottom and click "Create Token".

![Image1](content/posts/microsoft-sentinel-notifications/Picture1.png)

Give the token a name and make sure you copy the token before you close the pop-up as you will not be able to retrieve it at a later date.

![Image2](content/posts/microsoft-sentinel-notifications/Picture2.png)

Now we need to create a Home Assistant Script to carry out the actions that we want Home Assistant to carry out when a Sentinel Incident is raised.

![Image3](content/posts/microsoft-sentinel-notifications/Picture3.png)

In this script, I am calling WLED to carry out some specific colors based on the incident type as well as send a notification to my Discord Bot, this configuration can also be found on my GitHub.
Using the script method the possibilities are endless, you could send a notification to your mobile device, have voice notification to your virtual assistants such as Google Home, Amazon Alexa or Siri. I look forward to seeing what you come up with.

## Azure Sentinel Playbook Configuration

Let's look at how we create this Playbook.

In Azure Sentinel create a new Playbook (Logic App) and select the Trigger "When Azure Sentinel incident creation rule was triggered", then we need to create the API-Key to authenticate against Home Assistant.

You will also need to create a [managed identity](https://medium.com/r/?url=https%3A%2F%2Ftechcommunity.microsoft.com%2Ft5%2Fazure-sentinel%2Fwhat-s-new-managed-identity-for-azure-sentinel-logic-apps%2Fba-p%2F2068204) for this playbook that has the RBAC role of **Microsoft Sentinel Contributor**.

![Image4](content/posts/microsoft-sentinel-notifications/Picture4.png)

The Value of the "Initialize variable" should be in the following format:

```
Name: API-Key
Type: String
```

The Next action we need to create is "Run query and list results" using the query below. `You will need to authenticate this with an account that has access to Log Analytics workspace`.

```
let IncidentSeverity = dynamic(["High", "Medium", "Low"]);
SecurityIncident
| where Severity in~ (IncidentSeverity)
| project Severity
```

![Image5](content/posts/microsoft-sentinel-notifications/Picture5.png)

Next, we need to create a "For Each" loop, some "Conditions" for each of the Azure Sentinel severity types, and an "HTTP" action as outlined below:

![Image6](content/posts/microsoft-sentinel-notifications/Picture6.png)

Using the Home Assitant REST API we can POST to your instance of Home Assistant using the following URI:

```
https://yourinstanceofhomeassistant.ui.nabu.casa/api/services/script/turn_on
```

Be sure to add the following headers to the HTTP Request.

```
Authorization @variables('API-Key')
Content-Type application/json
```

Use the following JSON in the Body of the Request to make the REST call for the Home Assistant script we created earlier.

```JSON
{
  "entity_id": "script.sentinel_high"
}
```

You will need to duplicate this configuration for each of the Azure Sentinel incident severities. Make sure you change the "Condition" Severity and the "HTTP" Body to reflect the Home Assistant script for the severity type.

Finally, we just need to create an Azure Sentinel Automation to call the playbook each time an incident is raised in Azure Sentinel.
